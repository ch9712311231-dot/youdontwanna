# 원칙

> 사용자가 "이걸 원칙으로 만들어(해줘)"라고 명시적으로 말한 규칙만 이 파일에 기록한다.
> 등록된 원칙은 이후 모든 작업에서 항상 우선 적용한다.
> 사용자의 새 지시가 등록된 원칙과 어긋나면, 작업을 진행하기 전에 해당 원칙을 보여주고 어느 쪽을 따를지 먼저 확인한다.

형식:

```
## [번호]. 원칙 제목
- 내용: (구체적 규칙)
- 등록일: YYYY-MM-DD
- 배경: (왜 이 원칙이 생겼는지, 관련 대화 요약)
```

---

## 1. 같은 본문 내 자간(letter-spacing) 통일
- 내용: 같은 본문(박스 안 텍스트 등) 안에서는 볼드(strong)·일반 텍스트 구분과 무관하게 자간이 통일되어 보여야 한다. 볼드체가 원본 대비 더 빽빽해 보이면 `strong`에 소폭의 양수 letter-spacing(예: .012em)을 주어 시각적 밀도를 맞춘다.
- 등록일: 2026-07-11
- 배경: 03_ch01_exam-intro.html 박스 안에서 볼드(강조) 텍스트가 일반 텍스트보다 자간이 좁아 보인다는 지적. 실제 CSS letter-spacing 값은 모두 normal/0으로 동일했으나, Pretendard 폰트가 weight(700)에서 자간이 더 좁게 그려지는 특성 때문에 시각적으로 다르게 보였음 — `strong`에 letter-spacing:.012em을 추가해 보정.
