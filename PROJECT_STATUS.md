### 0-1. A5 + 2도인쇄 전환 완료 사양

**판형 (`src/book.css` `:root`)**

| 항목 | 값 |
|---|---|
| 트림 | 148×210mm (A5) |
| 도련 | 3mm → 작업 사이즈 154×216mm |
| 여백(위/아래/바깥/제본) | 13mm / 11mm / 11mm / 13mm (중철 기준) |
| 폰트 크기 | 기존 대비 ~90% 축소 (본문 8.6pt, 최소 라벨 6.8pt) |
| 간격(패딩·갭) | 기존 대비 ~72% 축소 |

**색상 — 2도인쇄 (검정 K + 브랜드 시안 #00ACC8)**

| 변수 | 값 | 용도 |
|---|---|---|
| `--ink` | `#181818` | 검정 100% — 본문 글자 |
| `--ink-soft` / `--muted` | `#4D4D4D` / `#8C8C8C` | 검정 tint — 보조/캡션 |
| `--brand` / `--brand-deep` | `#00ACC8` / `#017A93` | 시안 100% / 진한 tint — 포인트·표 헤더 |
| `--brand-ink` | `#0A4A5C` | 검정+시안 overprint — 제목·강조 |
| `--sky-50~300` | 시안 옅은 tint 4단계 | 카드/배경 |
| `--gray-50` / `--gray-200` | `#F2F2F2` / `#D6D6D6` | 검정 옅은 tint — 경고박스·감소카드 |

기존 amber(경고박스)·orange(감소카드·목업리본) 3곳은 색상 대신 **검정 tint + 아이콘("!" 배지)**으로 재설계해 2색 제약 안에서 의미를 구분함.

**검증**: `node build_book.js && node render.js` 로 실제 PDF 재생성 후 16페이지 전부 PNG 렌더링해 육안 검수 완료 — 잘림(overflow) 없음, 41개 과목 2단 표 등 밀도 높은 페이지도 A5 한 페이지에 정상 수납, 팔레트 이탈 색 없음. 표지 리본 캡션과 eyebrow 배지가 겹치던 문제 1건 발견 후 `.cover .inner` 상단 패딩(30→40mm)과 title/subtitle/desc 간격 조정으로 수정 완료.

**한계**: 이 파이프라인은 실제 인쇄소용 별색 분판(spot color separation) 파일을 만들지 않음 — 여기서 확정한 건 "검정+시안 2색조만 쓰는" 디자인 시안이며, 인쇄소 데이터 변환은 인쇄 단계에서 별도 협의 필요.

## 3. 표지 최종 사양

여러 라운드의 피드백(레퍼런스 이미지 2종 반영 → 전문 디자이너 관점 6가지 지시 → 로고 위치 재변경)을 거쳐 아래로 최종 수렴함:

- **배경**: 연한 하늘색 그라데이션 `linear-gradient(168deg, #F2FBFD 0%, #DEF4FA 45%, #BFE8F2 100%)` + 흰색/남색 톤 원형 데코 3개.
- **좌측 상단**: "1위 딱지" 리본 — 남색 그라데이션, 하단 제비꼬리 컷, "교원임용 / 1위 / 해커스". 바로 아래 3줄 근거문구(수동 줄바꿈): "2021 대한민국 No.1 대상 교원임용 교육 / (온·오프라인 교원임용) 부문 1위 / (한국미디어마케팅그룹)".
- **우측 상단**: 현재 비어 있음(로고를 하단으로 옮겼기 때문).
- **eyebrow**: "시작하는 중등 임용고시 수험생을 위한" — 흰 pill 배지, 13.5pt.
- **메인 타이틀**: 66pt, 굵게, 2줄 색 분리 — `해커스임용`(진한 남색) / `합격 로드맵`(브랜드 시안색, 로고 색과 동일).
- **서브타이틀 박스**: "중등 임용고시 대비" — 남색 배경, 흰 글씨 15pt, radius 2.2mm(덜 둥글게 조정됨).
- **설명 박스**: 2단 구성 — 메인 문구(13pt, 굵게) + 구분선 아래 서브 문구(10.5pt): "해커스임용 스타 강사진이 제시하는 과목별 합격 전략 / 중등 임용고시 최신 정보 수록".
- **하단(footer)**: 좌측 로고(흰 박스 없이 원본, 9mm 높이) / 우측 `teacher.Hackers.com`.
- **삭제된 요소**: "합격으로 가는 길" SVG 로드맵 그래픽(시작→경로→깃발)은 최종적으로 제거하고 타이포 확대로 대체.

## 6. 빌드 방법 / 작업 워크플로우

**원칙: 챕터 내용을 수정하는 동안은 해당 `src\parts\*.html` 파일만 건드리고, 최종본(`src\book.html`·PDF·전체 웹뷰어) 병합은 사용자가 "취합/최종본 만들어줘"라고 명시적으로 요청할 때만 진행한다.** (2026-07-06 확정 — 개발일지 참고)

- **부분 수정 후 바로 확인하고 싶을 때** (평소 작업 방식)
  ```bash
  node build_parts_preview.js   # src\parts\*.html 각각 → output\parts\*.html (챕터별 독립 미리보기, 더블클릭으로 열림)
  ```
  수정한 챕터의 `output\parts\<파일명>.html`만 다시 열어보면 됨. book.html·PDF·전체 웹뷰어는 건드리지 않음.

- **모든 수정이 끝나고 "최종본으로 취합해줘"라고 요청받았을 때만** 아래 3개를 순서대로 실행
  ```bash
  node build_book.js                             # src\parts\* → src\book.html 병합
  node render.js book.html <출력파일명>.pdf       # PDF 재생성 → output/ 폴더
  node build_web.js                              # 통합 웹 뷰어 재생성 → output/ 폴더
  ```

## 8. 알아둘 것 (버그·주의사항)

- 표 헤더(`<th>`)에 `.hl` 클래스를 쓰면 진한 배경과 겹쳐 글씨가 안 보이는 버그가 있었음 → th 강조는 밝은 색 인라인 스타일로.
- 레퍼런스 PDF는 `pdftotext`로 한글이 깨짐 → PyMuPDF(`_source/pdf_tool.py`)로 PNG 렌더링 후 시각적으로 읽어야 함(텍스트 추출 시 반드시 `open(file, 'w', encoding='utf-8')`로 파일 저장 — 터미널 출력은 한글이 깨짐).
- 웹뷰어 폰트는 현재 쓰인 문자만 추출한 서브셋 — PART 2 원고에서 서브셋에 없는 새 문자를 쓰면 웹뷰어에서만 글자가 깨질 수 있음(PDF는 안전). 문제 생기면 `fontTools.subset`으로 재서브셋 필요.
- **페이지 콘텐츠가 216mm를 넘으면 마지막 줄/항목이 티 안 나게 잘림(2026-07-11 발견)**: `.page{height:216mm; overflow:hidden; display:flex; flex-direction:column}` + `.folio{margin-top:auto}` 구조상 총 콘텐츠 높이가 216mm를 넘으면 flexbox 기본 `flex-shrink:1`이 `.blk` 박스를 강제로 눌러서 맞추고, `.blk`엔 `min-height`가 없어 초과분이 스크롤도 안 되고 에러도 없이 조용히 잘림. 표(`table.tbl`)나 리스트(`.plist2`)의 마지막 행/항목이 안 보이면 십중팔구 이 버그. **CSS 속성(table-layout, min-width 등)으로는 절대 못 고침** — 해결법은 (a) 내용을 줄여 한 페이지에 확실히 들어오게 하거나 (b) `<section class="page">`를 추가해 여러 페이지로 분할하는 것뿐(ch02·ch05·ch07·ch09가 이 방식으로 여러 페이지 사용 중). 새 챕터 작성 후엔 항상 실제 렌더링(스크린샷 또는 `scrollHeight` vs `clientHeight` 비교 스크립트)으로 잘림 여부를 확인할 것.

## 0-2. 실물 교재 미리보기(스프레드) 전환 — ✅ 완료 (2026-07-09)

사용자가 "실제 교재느낌을 살리면서 두 페이지씩 보이게" 요청 + 참고 샘플 전달(민범팟 합격선 포켓북 웹뷰어). 확인 결과 아래 항목은 이미 이전(미커밋) 세션에서 구현되어 있었고, 이번 세션에 표지/목차 단독 페이지 처리만 샘플 관례에 맞춰 보완함:

- **화면 스프레드 뷰**: `build_web.js`가 `output\임용합격로드맵_web.html`을 두 페이지씩(`.spread`) 짝지어 생성. 제본 안쪽 여백(gutter) 자동 좌우 전환 + 펼침면 안쪽 그림자(`box-shadow: inset`)로 실물 느낌 구현. **PDF(render.js) 출력에는 영향 없음** — book.html 자체엔 `.spread` 래퍼가 없어 순차 낱장 그대로 인쇄됨.
- **표지 인쇄 임포지션**: `build_cover_spread.js` → `output\표지_스프레드.pdf`(뒤표지-앞표지 한 장, 308×216mm) + `output\표지_스프레드_미리보기.html`(화면용). 본문 시퀀스와 별개의 독립 산출물.
- **뒤표지**: `src\parts\back-cover.html` 신규 추가(임시 디자인 — 로고+카피+ISBN 자리, 실제 문구 도착 시 교체 예정).
- **단독 페이지 recto 시작 규칙** (이번 세션 추가, `book.css` `.page.blank` + `build_web.js`): 표지(1p)·목차(2p)는 실제 인쇄 관례상 왼쪽에 빈 면을 넣어 오른쪽(recto)에서 단독 시작하도록 처리. 화면 미리보기 전용 — PDF엔 영향 없음. 파트 간지(챕터 도입부)는 빈 면 삽입 없이 바로 다음 페이지와 자연스럽게 짝지음(샘플과 동일 관례).

## 0-4. 표지·목차 디자인 리터치 (2026-07-09)

- **뒤표지 재설계**: `src/parts/back-cover.html` — 텍스트(로고·타이틀·바코드·연락처)를 책등 쪽(스프레드에서 오른쪽 안쪽)으로 몰아 비대칭 배치. 이후 사용자가 실물 참고 사진(`임용 ATOZ.jpg`, `임용 합격로드맵.jpg`) 2종을 전달해 타이틀 3줄 배치("해커스임용/합격/로드맵", 앞표지와 동일한 색상 로직), 서브타이틀 박스("중등 임용고시 대비" — 앞표지 박스 축소 재사용), 본문 카피(참고 이미지 문구 그대로, 특정 문장 볼드+마침표+문단 간격 지정)까지 반영. 로고 크기는 앞표지 기준(7mm)으로 통일.

- **목차 그룹 헤더**: "PART 1/2" 표기를 "해커스임용 / 합격 로드맵 01·02"(원본 자료와 동일한 라벨 방식)로 변경, 깃발 아이콘 + 설명 텍스트 + 언더라인 형태로 재구성. PART2 과목 목록의 "—" 구분 기호 제거("교육학 합격 전략" 등 심플 표기).
- **주의**: 표지(`00_cover.html`)나 목차를 고칠 때 `build_web.js`만 돌리면 화면 미리보기의 "표지 임포지션" 블록엔 반영되지만, 본문 덱에 쓰이는 `src/book.html`은 `build_book.js`를 별도로 돌려야 갱신됨 — 이번 세션에서 이걸 빠뜨려 "반영이 안 된다"는 혼선이 한 번 있었음. **표지/목차/본문 아무 파트나 고친 뒤에는 `build_book.js` → `build_web.js` 순서로 항상 같이 돌릴 것.**
- 신규 이미지 자산은 `assets/img/`에 저장(로고 전용이던 `assets/logo/`와 별도 폴더). `build_web.js`/`build_parts_preview.js`의 이미지 인라인 정규식을 `assets/(logo|img)/...` 둘 다 매칭하도록 확장함.
